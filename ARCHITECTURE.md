## Security Flow

```
┌─────────────────────────────────────────────────────────────────┐
│                    Security Layers                              │
│                                                                  │
│  Layer 1: Email Verification                                   │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │ • Email must match registered account                     │ │
│  │ • Wrong email → No reset email sent (for privacy)        │ │
│  │ • Acts as first security check                           │ │
│  └───────────────────────────────────────────────────────────┘ │
│                           ▼                                     │
│  Layer 2: Secure Token Generation                             │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │ • Firebase generates cryptographically secure token       │ │
│  │ • Token embedded in email reset link                      │ │
│  │ • Token unique per request (not predictable)             │ │
│  │ • Token already "tied" to firebase auth system           │ │
│  └───────────────────────────────────────────────────────────┘ │
│                           ▼                                     │
│  Layer 3: Time-Based Expiration                               │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │ • Reset links valid for 1 hour only                       │ │
│  │ • Timestamp checked on every password update             │ │
│  │ • Expired links rejected automatically                    │ │
│  └───────────────────────────────────────────────────────────┘ │
│                           ▼                                     │
│  Layer 4: HTTPS Encryption                                    │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │ • All data transmitted over encrypted HTTPS              │ │
│  │ • Password never sent as plain text                       │ │
│  │ • Man-in-the-middle attack prevention                     │ │
│  └───────────────────────────────────────────────────────────┘ │
│                           ▼                                     │
│  Layer 5: Firebase Auth Database                              │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │ • Passwords hashed with bcrypt                            │ │
│  │ • Never stored as plaintext                               │ │
│  │ • Google's enterprise security                            │ │
│  │ • Automatic failover & backups                            │ │
│  └───────────────────────────────────────────────────────────┘ │
│                           ▼                                     │
│  Layer 6: Rate Limiting                                       │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │ • Limits password reset requests (prevents brute-force)   │ │
│  │ • Rate: ~5 requests per user per hour                     │ │
│  │ • Automatic cooldown after too many attempts             │ │
│  └───────────────────────────────────────────────────────────┘ │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```
